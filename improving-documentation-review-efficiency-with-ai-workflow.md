# Improving documentation review efficiency with AI-assisted workflow

After writing the docs, there was usually an empty window of waiting for the SMEs review them for technical accuracy and clarity. I would usually get a response as we approached the release date. Sicne docs were a part of the release process, it created a bottleneck situation wherein I would need to accommodate any last-minute design changes or other changes that require last-minute churn. This also led to trade-offs and prioritization issues. 

## Context
To collaborate at AWS, [Quip](https://quip.com/) was widely used to create and edit document by multiple collaborators. Quip was heavily used for design documents, new product ideas, meeting notes, and project design and deliverables.   
The workflow that I was following for the review was converting the local HTML build to generate a PDF, highlighting updates in the PDF with prefixes as "new", "review here", or "updated", and then eventually uploading it to Workdocs.
